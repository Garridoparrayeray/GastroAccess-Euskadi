### 🎯 Objetivo del Proyecto

El objetivo es idear una solución digital basada en datos reales. Buscamos un **concepto sólido, validado por datos y bien comunicado**.

No se pide una aplicación compleja; tenéis pocas horas. Se pide inteligencia en el uso de los datos, creatividad en la propuesta y orden en la ejecución. **Think Big, Start Small.**

---

### ⚙️ Flujo de Trabajo y Entregas (Git Flow)

La gestión del trabajo y las entregas se realizarán exclusivamente a través del repositorio, simulando un entorno profesional real mediante **Pull Requests (PR)** contra la rama `main` (o `master`).

**1. Entrega de la Propuesta (El Semáforo):**

* Debéis crear un archivo `PROPUESTA.md` en vuestro repositorio.
* Haced una **Pull Request** a la rama principal con este archivo.
* **Aprobación:** Yo revisaré la PR. Si la idea es viable, aceptaré (merge) la PR. Ese es vuestro "semáforo en verde" para empezar. Si no, os pediré cambios en los comentarios de la PR.

**2. Entrega Final del Trabajo:**

* Fecha límite de entrega (Código/Documentación): **18 de febrero (antes de las 23:59)**.
* Todo el trabajo debe estar subido mediante una nueva Pull Request o commit final en la rama principal antes de esta fecha.

**3. Presentación Oral:**

* Fecha: **23 de febrero** (en clase).

---

### 📋 Requisitos del Proyecto

1. **Datos Reales:**
* **Fuente:** Open Data Euskadi (Prioritario) o Web Scraping.
* **Evidencia:** El repositorio debe incluir el script de extracción o el dataset "crudo" y el dataset "limpio".


2. **Prototipo:** Un MVP (Producto Mínimo Viable) que demuestre la idea. Puede ser un notebook de análisis, una web estática sencilla o un prototipo interactivo (Figma/Penpot) acompañado de los datos que lo alimentan.
3. **Documentación:** El `README.md` del repositorio debe estar impecable, explicando qué es el proyecto, de dónde salen los datos y cómo se interpretan.

---

### ⚖️ Rúbrica de Evaluación

La nota se dividirá priorizando la capacidad de investigación, organización y comunicación.

| Criterio | Peso | Descripción |
| --- | --- | --- |
| **Investigación y Uso del Dato** | **35%** | ¿La fuente de datos es interesante? ¿Los datos son relevantes para el problema planteado? ¿Se ha realizado una limpieza/extracción correcta (aunque sea sencilla)? |
| **Comunicación y Presentación** | **30%** | Claridad en la exposición oral (storytelling), calidad de las diapositivas y capacidad de síntesis. ¿Vende bien la idea? |
| **Organización y Metodología** | **20%** | Uso correcto del repositorio (commits claros, estructura de carpetas), cumplimiento de fechas (entrega del 18 feb) y flujo de la Pull Request. Documentación (`README`). |
| **Prototipo Técnico** | **15%** | Que el prototipo (código o visualización) funcione lo suficiente para entender la idea. No se evalúa la complejidad del código, sino que sirva para demostrar el concepto. |

### 📅 Cronograma (4 Semanas)

Solo tenemos 2 horas presenciales por semana. Aprovechadlas al máximo.

* **Semana 1:** Exploración de Open Data Euskadi. Redacción del archivo `PROPUESTA.md` y envío de la **Pull Request**.
* **Semana 2:** Obtención y limpieza de datos. Definición del alcance del prototipo.
* **Semana 3:** Desarrollo del prototipo y preparación de la documentación en el repo.
* **Semana 4 (Hasta el día 18):** Finalizar detalles, subir todo al repo (Deadline día 18). Preparar la presentación para el día 23.
* **Día 23 de febrero:** Presentación final en clase.

---

### 📝 Estructura de la Propuesta (`PROPUESTA.md`)

Para que os apruebe la PR rápidamente, el archivo debe contener:

1. **Título del Proyecto.**
2. **Descripción del problema:** (2-3 líneas).
3. **Origen de los datos:** (URL exacta del dataset o web a scrapear).
4. **Valor aportado:** ¿Qué información nueva o servicio obtenemos al procesar esos datos?

### 💡 Ideas para inspiraros (Ejemplos)

* **Turismo Inteligente:** Usar datos de "Alojamientos turísticos en Euskadi" para crear un mapa de zonas saturadas vs. zonas tranquilas.
* **Movilidad:** Usar datos de tráfico en tiempo real o parkings públicos para predecir la mejor hora para ir al centro.
* **Cultura:** Scrapear la agenda cultural de un ayuntamiento para sugerir planes de "último minuto" gratuitos.
* **Meteorología y Salud:** Cruzar datos de polen/calidad del aire con ubicaciones de parques para recomendar dónde hacer deporte hoy.
