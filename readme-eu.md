### 🎯 Proiektuaren Helburua

Helburua benetako datuetan oinarritutako irtenbide digital bat asmatzea da. **Kontzeptu sendoa, datuekin balidatua eta ondo komunikatua** bilatzen dugu.

Ez da aplikazio konplexu bat eskatzen; ordu gutxi dituzue. Datuen erabileran adimena, proposamenean sormena eta exekuzioan ordena eskatzen da. **Think Big, Start Small.**

### ⚙️ Lan-fluxua eta Entregak (Git Flow)

Lanaren kudeaketa eta entregak biltegiaren (repository) bidez bakarrik egingo dira, ingurune profesional erreal bat simulatuz, `main` (edo `master`) adarraren aurkako **Pull Requests (PR)** erabiliz.

**1. Proposamenaren Entrega (Semaforoa):**

* `PROPUESTA.md` fitxategi bat sortu behar duzue zuen biltegian.
* Egin **Pull Request** bat adar nagusira fitxategi honekin.
* **Onarpena:** Nik PRa berrikusiko dut. Ideia bideragarria bada, PRa onartuko (merge) dut. Hori da hasteko zuen "semaforo berdea". Ez bada bideragarria, aldaketak eskatuko dizkizuet PRaren iruzkinetan.

**2. Lanaren Amaierako Entrega:**

* Entregatzeko epemuga (Kodea/Dokumentazioa): **Otsailak 18 (23:59ak baino lehen)**.
* Lan guztia igo behar da Pull Request berri baten bidez edo azken commit baten bidez adar nagusian data hori baino lehen.

**3. Ahozko Aurkezpena:**

* Data: **Otsailak 23** (klasean).

### 📋 Proiektuaren Baldintzak

1. **Benetako Datuak:**

* **Iturria:** Open Data Euskadi (Lehentasunezkoa) edo Web Scraping.
* **Ebidentzia:** Biltegiak erauzketa-scripta edo dataset " gordina" (raw) eta dataset "garbia" barne hartu behar ditu.

2. **Prototipoa:** Ideia erakusten duen MVP (Bideragarritasun Gutxieneko Produktua) bat. Azterketa-notebook bat, web estatiko sinple bat edo prototipo interaktibo bat (Figma/Penpot) izan daiteke, elikatzen duten datuekin lagunduta.
3. **Dokumentazioa:** Biltegiko `README.md` fitxategiak ezin hobea izan behar du, proiektua zer den, datuak nondik datozen eta nola interpretatzen diren azalduz.

### ⚖️ Ebaluazio-errubrika

Nota banatzerakoan, ikerketa-gaitasuna, antolakuntza eta komunikazioa lehenetsiko dira.

| Irizpidea | Pisua | Azalpena |
| --- | --- | --- |
| **Ikerketa eta Datuen Erabilera** | **35%** | Datu-iturria interesgarria al da? Datuak garrantzitsuak al dira planteatutako arazorako? Garbiketa/erauzketa zuzena egin al da (sinplea bada ere)? |
| **Komunikazioa eta Aurkezpena** | **30%** | Ahozko azalpenaren argitasuna (storytelling), diapositiben kalitatea eta sintesi-gaitasuna. Ideia ondo saltzen al du? |
| **Antolakuntza eta Metodologia** | **20%** | Biltegiaren erabilera zuzena (commit argiak, karpeten egitura), daten betetzea (otsailaren 18ko entrega) eta Pull Request fluxua. Dokumentazioa (`README`). |
| **Prototipo Teknikoa** | **15%** | Prototipoak (kodea edo bisualizazioa) ideia ulertzeko nahikoa funtzionatzea. Ez da kodearen konplexutasuna ebaluatzen, baizik eta kontzeptua erakusteko balio izatea. |

### 📅 Kronograma (4 Aste)

Astean 2 ordu presentzial baino ez ditugu. Atera iezaiezue ahalik eta etekin handiena.

* **1. Astea:** Open Data Euskadiren esplorazioa. `PROPUESTA.md` fitxategia idaztea eta **Pull Request**-a bidaltzea.
* **2. Astea:** Datuak lortzea eta garbitzea. Prototipoaren irismena zehaztea.
* **3. Astea:** Prototipoaren garapena eta dokumentazioa prestatzea biltegian.
* **4. Astea (Hilaren 18ra arte):** Xehetasunak amaitu, dena biltegira igo (Deadline: hilak 18). Hilaren 23ko aurkezpena prestatu.
* **Otsailak 23:** Amaierako aurkezpena klasean.

PRa azkar onartzeko, fitxategiak honako hau eduki behar du:

1. **Proiektuaren Izenburua.**
2. **Arazoaren deskribapena:** (2-3 lerro).
3. **Datuen jatorria:** (Datasetaren URL zehatza edo scrapeatu beharreko webgunea).
4. **Ekarritako balioa:** Zer informazio berri edo zerbitzu lortzen dugu datu horiek prozesatzean?

### 💡 Inspiratzeko Ideiak (Adibideak)

* **Turismo Adimenduna:** "Euskadiko turismo-ostatuak" datuak erabiltzea, eremu saturatuen eta eremu lasaien mapa bat sortzeko.
* **Mugikortasuna:** Trafikoari buruzko denbora errealeko datuak edo aparkaleku publikoenak erabiltzea, erdigunera joateko ordu onena iragartzeko.
* **Kultura:** Udal baten kultur agenda scrapeatu, "azken orduko" doako planak iradokitzeko.
* **Meteorologia eta Osasuna:** Polenaren/airearen kalitatearen datuak parkeen kokalekuekin gurutzatzea, gaur kirola non egin gomendatzeko.
